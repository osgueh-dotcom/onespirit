# PRODUCT_VISION.md

## Product Name

OneSpirit Workflow System

---

## Vision

OneSpirit Workflow System should become the central operating system for PT One Spirit Asia project and event workflow.

It should help the team manage event work from inquiry to final report with clearer status, better document control, stronger operational readiness, and better financial visibility.

---

## Product Positioning

OneSpirit is not just an admin dashboard.

It is a business workflow system for event operations.

The product should help users answer:

```txt
What project are we handling?
What stage is it in?
Is the CL ready?
Is the ROS ready?
Is the CK complete?
Is the PNL under control?
What is missing?
What is the next action?
Who is responsible?
Is the project ready for execution?
```

---

## Core Product Promise

The system should reduce confusion, manual checking, and scattered coordination.

It should make project readiness visible.

---

## Product Experience Principles

### 1. Workflow-first

Every screen should help users move work forward.

### 2. Clear status

Users should quickly understand whether something is:

```txt
Not Started
In Progress
Blocked
Ready
Completed
Needs Review
```

### 3. Clear next action

The system should guide users toward the next useful step.

Examples:

```txt
Create CL
Complete ROS
Review CK
Finalize PNL
Upload missing document
Confirm project details
```

### 4. Missing data visibility

Missing important data should not be hidden.

### 5. Professional simplicity

The UI should be clean, calm, and business-oriented.

### 6. Traceability

Important changes should be traceable over time.

## Product Quality Bar

A feature is good enough only if:

```txt
[ ] It supports real workflow
[ ] The terminology is correct
[ ] The user knows what to do next
[ ] Important missing data is visible
[ ] It works without breaking existing pages
[ ] It can be extended later
[ ] It is validated before commit
```

---

## Innovation Direction

Innovation should focus on practical business value:

- Automatic next action
- Missing document detection
- Readiness score
- Timeline risk warning
- Auto-generated CK from project type
- ROS template
- PNL summary
- Project archive package
- Management overview
- Operational command center

Avoid innovation that is only decorative.

---

## Product North Star

The north star is:

```txt
Every OneSpirit project should have a clear status, clear required documents, clear operational readiness, clear financial position, and clear next action.
```
