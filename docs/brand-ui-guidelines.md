# Brand UI Guidelines - OneSpirit Workflow

Dokumen ini merangkum arah visual Sprint 14.1 untuk menjaga tampilan OneSpirit tetap clean, profesional, dan konsisten.

## 1. Brand Color Direction

- Primary brand: orange One Spirit dari logo.
- Secondary: amber / yellow untuk aksen dan highlights.
- Neutral: white, off-white, light gray, dan charcoal untuk teks.
- Semantic:
  - green untuk success / confirmed
  - orange untuk pipeline / brand highlight
  - blue untuk info / comparison
  - red untuk cancel / danger

## 2. UI Tone

- Modern, energetic, digital, dan corporate-operational.
- Clean lebih penting daripada dekoratif.
- Setiap halaman harus mudah dipindai dalam 5 detik pertama.
- Dashboard harus informatif, bukan sekadar visual hiasan.

## 3. Logo Usage

- Gunakan logo One Spirit di login page, app shell, dan area brand yang relevan.
- Jaga proporsi logo agar tidak mendominasi layar.
- Hindari pengulangan logo berlebihan dalam satu halaman.
- Logo harus selalu punya ruang kosong yang cukup.

## 4. Component Styling Principles

- Pakai card putih atau off-white dengan border ringan.
- Shadow harus halus.
- Button harus konsisten:
  - primary untuk aksi utama
  - secondary untuk aksi pendukung
  - danger untuk aksi destruktif
- Badge harus ringkas dan mudah dibaca.
- Typography harus punya hierarchy yang jelas: title, subtitle, body, meta.

## 5. Chart Color Rules

- Orange / amber untuk pipeline, brand highlight, dan progress.
- Green untuk confirmed / success.
- Red untuk cancel / risk.
- Blue untuk comparison dan insight informasional.
- Hindari terlalu banyak warna dalam satu chart.

## 6. Light Mode Principles

- Background utama terang dan lembut.
- Table header dan filter panel harus konsisten.
- Alert gunakan tinted background yang ringan, bukan panel gelap berat.
- Kontras teks harus cukup untuk dibaca saat Zoom share atau di mobile.

## 7. Known Limitations

- Beberapa halaman lama masih memakai utility class lama dan akan dirapikan bertahap.
- Dark mode masih tersedia, tetapi default experience diarahkan ke light mode.
- Brand polish ini tidak mengubah backend business logic.
- Dashboard analytics tetap bergantung pada kualitas data yang tersedia.
